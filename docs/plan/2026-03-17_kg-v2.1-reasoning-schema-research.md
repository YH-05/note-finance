### Step A: research-neo4j スキーマ設計変更

#### A-1: AUTHORED_BY リレーション追加 (P0)

Source.publisher フィールドから Author ノードを実体化し、AUTHORED_BY (Source → Author) を生成。
現在の Author は LLM 抽出の `author_name` ベースだが、research-neo4j では Source.publisher からの実体化を**併用**する。

**影響ファイル**:
- `scripts/emit_graph_queue.py` — `_build_authored_by_rels(sources, author_name_to_id)` 追加、RELATION_KEYS に `authored_by` 追加
- `data/config/knowledge-graph-schema.yaml` — AUTHORED_BY 定義（既に存在するが emit 側が未対応）
- `.claude/skills/save-to-graph/guide.md` — AUTHORED_BY Cypher テンプレート追加
- テスト追加

**設計詳細**:
- `map_pdf_extraction()` の後処理で Source.publisher → Author ノード生成
- LLM 抽出 Author と publisher Author は `author_name_to_id` で統合重複排除
- Source の publisher フィールドが空の場合はスキップ

#### A-2: consensus_divergence question_type 追加 (P2)

research-neo4j 固有の Question タイプ。複数アナリスト間のレーティング乖離を検出。

**影響ファイル**:
- `src/pdf_pipeline/schemas/extraction.py` — `ExtractedQuestion.question_type` Literal に追加
- `data/config/knowledge-graph-schema.yaml` — question_type enum に追加
- `src/pdf_pipeline/core/knowledge_extractor.py` — `_EXTRACTION_PROMPT` に説明追加

### Step B: 両DB共通改修 (AUTHORED_BY + TREND metric_id)

#### B-1: AUTHORED_BY を article-neo4j にも適用

article-neo4j では Source.publisher が空のケースが多いが、存在する場合は AUTHORED_BY を生成。
emit_graph_queue.py の改修は Step A と共通。

#### B-2: `_load_metric_alias_index()` 追加

`data/config/metric_master.json` のエイリアス→metric_id ルックアップを構築し、TREND グルーピングに使用。

**影響ファイル**:
- `scripts/emit_graph_queue.py`:
  - `_load_metric_alias_index()` — `metric_master.json` からエイリアス辞書構築（`apply_metric_master.py` のロジック流用）
  - `_build_trend_edges()` — グルーピングキーを `(ticker, metric_name)` → `(ticker, metric_id)` に変更。metric_id 未解決の DP はフォールバックで metric_name 使用
- テスト — `TestLoadMetricAliasIndex` 追加

#### B-3: TREND に metric_id プロパティ付与

TREND リレーションに `metric_id` を追加し、Metric ノード経由せずに同一指標チェーンを辿れるようにする。

**影響ファイル**:
- `scripts/emit_graph_queue.py` — `_build_trend_edges()` の出力に `metric_id` 追加
- `data/config/knowledge-graph-schema.yaml` — TREND プロパティに `metric_id` 追加
- `.claude/skills/save-to-graph/guide.md` — TREND Cypher に `metric_id` 追加

#### B-4: MEASURES 未リンク DP は TREND 対象外（research-neo4j のみ）

research-neo4j では MEASURES リレーションが存在するため、未リンク DP の metric_name 比較は誤解を招く。
`_build_trend_edges()` に `measures_linked_dp_ids: set[str] | None` パラメータ追加。None（article-neo4j）の場合は全 DP 対象。

---

### Step C: 遡及バッチスクリプト

#### C-1: `scripts/backfill_stance_from_claims.py` (新規)

既存 Claim(732件) のうち rating/target_price 付きのものから Stance を遡及生成。

**ロジック**:
1. research-neo4j から `MATCH (c:Claim) WHERE c.rating IS NOT NULL OR c.target_price IS NOT NULL RETURN c` で取得
2. Claim.source_id → Source.publisher で Author 特定
3. Claim → Entity (ABOUT) で entity 特定
4. Stance + HOLDS_STANCE + ON_ENTITY + BASED_ON を生成
5. 同一 (author, entity) で SUPERSEDES チェーン構築
6. `--dry-run` モードで件数レポート

#### C-2: `scripts/backfill_temporal_chain.py` (新規)

既存 FinancialDataPoint(166件) / FiscalPeriod(16件) から NEXT_PERIOD + TREND を遡及生成。

**ロジック**:
1. research-neo4j から全 FP / DP を取得
2. `_period_sort_key()` で FP ソート → NEXT_PERIOD 生成
3. MEASURES リンク済み DP のみ対象 → metric_id でグルーピング → TREND 生成
4. `--dry-run` モードで件数レポート

## 検証方法

```bash
# Phase 2 共通
make check-all
uv run pytest tests/scripts/test_emit_graph_queue.py -v

# graph-queue 生成テスト
uv run python scripts/emit_graph_queue.py --command pdf-extraction --input data/processed/HSBC_ISAT/

# research-neo4j 投入テスト (port 7688)
docker start research-neo4j
NEO4J_URI=bolt://localhost:7688 /save-to-graph --file .tmp/graph-queue/pdf-extraction/gq-*.json

# AUTHORED_BY 検証
cypher-shell -a bolt://localhost:7688 -u neo4j -p "$NEO4J_PASSWORD" \
  'MATCH (s:Source)-[:AUTHORED_BY]->(a:Author) RETURN a.name, count(s) ORDER BY count(s) DESC'

# TREND metric_id 検証
cypher-shell -a bolt://localhost:7688 -u neo4j -p "$NEO4J_PASSWORD" \
  'MATCH ()-[r:TREND]->() WHERE r.metric_id IS NOT NULL RETURN r.metric_id, count(r), avg(r.change_pct)'

# 遡及バッチ検証
uv run python scripts/backfill_stance_from_claims.py --dry-run
uv run python scripts/backfill_temporal_chain.py --dry-run
```

## AI推論パス（Phase 2 で有効化されるクエリ）

```cypher
-- AUTHORED_BY: ソースとアナリストの紐付け
MATCH (s:Source)-[:AUTHORED_BY]->(a:Author)-[:HOLDS_STANCE]->(st:Stance)-[:ON_ENTITY]->(e:Entity)
RETURN a.name, e.ticker, st.rating, st.target_price, s.title

-- TREND metric_id: 正規化指標での推移追跡
MATCH (d1)-[:TREND {metric_id: "metric-revenue"}]->(d2)-[:TREND {metric_id: "metric-revenue"}]->(d3)
MATCH (d1)-[:RELATES_TO]->(e:Entity)
RETURN e.ticker, d1.value, d2.value, d3.value

-- consensus_divergence: アナリスト間の見解乖離検出
MATCH (q:Question {question_type: "consensus_divergence"})-[:ASKS_ABOUT]->(e:Entity)
RETURN q.content, e.ticker, q.priority
```
