# 議論メモ: 副業・資産形成テーマの事例分析型への方針転換

**日付**: 2026-03-09
**参加**: ユーザー + AI

## 背景・コンテキスト

副業プロジェクトでは合成パターン法（5-10件のリアル体験を1つの架空人物に合成する手法）で
体験談DB記事を制作していたが、副業・ビジネス系テーマでは「架空のエピソード」よりも
「実際にうまくいった事例をまとめる」方が読者にとって有益ではないかという提案があった。

## 議論のサマリー

### 論点1: 合成パターン法 vs 事例分析型
- 合成パターン法: 架空の一人称ストーリー「僕は27歳で副業を始めた…」
- 事例分析型: 分析者視点「月5万稼いだ人の共通点は…」
- **結論**: 副業・資産形成は実用性が求められるため事例分析型が適切

### 論点2: 適用範囲
- 副業＋資産形成 → 事例分析型に切り替え
- 婚活 → 合成パターン法を継続（感情共感がコアバリュー）

### 論点3: 記事形式
- ミックス型を採用: ジャンル別の事例整理 + ジャンル横断の共通点抽出
- 副業例: 「Webライティング副業で月5万達成した人の戦略」+ 「副業で月5万超えた人の共通点」
- 資産形成例: 「つみたてNISAで3年続けた人の運用実績」+ 「投資初心者が100万貯めた共通ステップ」

### 論点4: ソース
- 既存パイプライン（Reddit + note.com + Web検索）で開始
- X・YouTube等は必要に応じて後から追加

### 論点5: AI活用一人スタートアップ事例の追加
- 最近AIを活用して一人スタートアップで大きな成功を収める事例が海外中心に増加
- これらの成功事例を分析・まとめることを副業テーマのスコープに追加
- 前回の「AI副業ハウツー除外」（dec-niche-001）とは別物: ツール紹介ではなくビジネス戦略分析
- 副業テーマ内の**看板シリーズ（注力コンテンツ）**として位置づけ
- ソース: Indie Hackers, Hacker News, Reddit (r/SideProject等) の英語圏コミュニティ

## 決定事項

1. **副業＋資産形成を事例分析型に方針転換**: 架空一人称 → 実在事例の成功パターン分析
2. **婚活は合成パターン法を継続**: 感情共感がコアなので現行アプローチが最適
3. **記事形式はミックス型**: ジャンル別整理 + ジャンル横断共通点抽出
4. **ソースは既存パイプラインで開始**: Reddit + note.com + Web検索
5. **AI活用一人スタートアップ事例を副業テーマの看板シリーズに追加**: 海外事例中心のビジネス戦略・成功パターン分析。AIハウツーではなくビジネスモデル分析が主眼（dec-niche-001のAI副業除外とは別カテゴリ）
