# 

■ なぜ、彼は「自分の意見」を書くのをやめたのか？

ここに、Pat Walls（パット・ウォールズ）という男がいます。

[

![画像](https://pbs.twimg.com/media/HBBilXqa8AE7m97?format=jpg&name=medium)







](/ceo_tommy1/article/2022246626765734324/media/2022224626835976193)

彼の運営するサイト「Starter Story」は、個人ブログからスタートしましたが、今や「ビジネス系メディアの巨人」となっています。

[

![画像](https://pbs.twimg.com/media/HBBilX5aQAI981n?format=jpg&name=medium)







](/ceo_tommy1/article/2022246626765734324/media/2022224626898845698)

掲載されているのは、4,000件以上の「一般人がどうやって起業し、いくら稼いだか」という「収益と戦略のデータベース」です。

売上データから集客方法まで、すべての答えが丸裸にされたこのサイトが叩き出している数字を見てください。

-   月間訪問者数： 160万人以上
    
-   掲載事例数： 4,000件以上の成功事例データベース
    
-   メール購読者： 12万人以上のアクティブ読者
    
-   運営体制： 7人のチームで運営
    
-   年間収益： $1.1M（約1.6億円）
    

[

![画像](https://pbs.twimg.com/media/HBBilYBb0AAuV7b?format=jpg&name=medium)

ここで鋭い方は気づくはずです。

「年商1.6億円で7人？ そんなに儲かってなくないか？」と。

しかし、これこそがこのモデル最大の「罠」であり「旨味」です。普通のメディア企業なら、7人のスタッフは「高給取りの編集者」や「カメラマン」です。 しかし、Starter Storyの7人は違います。

彼らは、パット・ウォールズが作ったマニュアル通りにデータを入力・整理する「運営スタッフ」です。

-   クリエイティブ不要： 誰でもできる作業なので、人件費は極小。
    
-   在庫ゼロ： 店舗も仕入れもありません。
    

つまり、売上のほぼ全額（利益率90%以上）が、そのまま「利益」として残る構造なのです。

> 「自分が寝ている間に、月1,000万円以上の現金が自動で振り込まれ続ける」

この「完全なる自動販売機」を持っていることこそが、数字以上に恐ろしい彼の本当の凄みなのです。

しかし、彼も最初からこうだったわけではありません。 むしろ最初は、今の規模とは程遠い「普通の開発者ブロガー」でした。彼は当初、自分のプログラミング論や、起業への想いを綴っていました。

しかし、アクセス解析を見た時、ある「事実」に気づきます。

-   自分の意見を書いた記事： 公開直後は読まれるが、すぐにタイムラインから消え、読まれなくなる。
    
-   他人の成功事例をまとめた記事： 公開直後の反応は薄いが、1ヶ月後も、半年後も検索からアクセスが入ってくる。
    

彼は悟りました。

> 「読者は『俺の意見』には興味がない。興味があるのは『他人がどう成功したかという事実』だけだ」

-   普通のブロガー視点： 「ブログで稼ぐには、継続が大事だと思います」→ 意見・価値が低い
    
-   パット・ウォールズの視点： 「この人は、3年間毎日更新して、月50万稼ぎました」→ 価値が高い
    

彼は「著者」として振る舞うのをやめ、「事例を集めて並べる整理役」に徹しました。 自分の主張を捨て、世の中にある「実例」を集めることに特化した瞬間、彼のサイトは個人のブログを超え、「事例が増えるほど強くなるデータベース」へと進化したのです。

> トミー： 多くの人は「自分が有名にならないと稼げない」と勘違いしています。しかし、パット・ウォールズは逆です。彼は個人の主張ではなく、「実例の蓄積」でサイト自体の信頼を作りました。 自分が前に出なくても構いません。「凄い人たち」が一番集まっている場所を作れば、その場所自体の価値が、中の誰よりも高くなるのです。

# 

■ でも、「自分には何もない」と思っていませんか？

ここまで読んで、あなたはおそらくこう感じているはずです。

-   「理屈はわかる。でも、それは彼が優秀だからできたんでしょ？」
    
-   「自分には、取材できるような起業家の知り合いなんて一人もいない」
    

その感覚は正しいです。 しかし、ここで勘違いしてはいけないのは、パット・ウォールズが勝った理由が「人脈」や「才能」だと思ってしまうことです。

彼が実際に強かったのは、個人の能力ではなく、「情報の積み上がり方の設計」でした。

多くの発信者が苦しむ中で、なぜ彼だけが資産を築けたのか？ その違いは、以下の3点に集約されます。

1.  情報の寿命が違う 「意見」は投稿した瞬間にタイムラインから流れて消えますが、「事例」は検索され続け、残り続けます。
    
2.  比較ができる 質問フォーマットが統一されているため、読者は複数の事例を「横並びで比較」することができ、そこに辞書のような価値が生まれます。
    
3.  数が増えるほど「自分事」になる 事例の数が増えれば増えるほど、読者が「自分と全く同じ境遇の成功者」を見つけられる確率が上がり、サイトの価値が指数関数的に高まります。
    

つまり彼は、書けば書くほど消耗する「労働」の形を捨て、集めれば集めるほど価値が増す「資産」の形に変えたのです。

ここが腹落ちすると、次の章で解説する「フローとストック」の意味がズレません。

-   なぜ、毎日投稿しても楽にならないのか？
    
-   なぜ、事例を資産として積む方が、凡人にとっての勝ち筋なのか？
    

その決定的な「構造の違い」を、第1章で解説します。

# 

■ 第1章：「投稿」を書くな「データ」を貯めろ

多くの発信者は、SNSやブログで毎日「流れて消えるフロー情報」を書いて消耗しています。 しかし、パット・ウォールズはサイトを「データベース」として設計しました。

彼が集めているのは、4,000人以上の起業家の「スペック表」です。

[

![画像](https://pbs.twimg.com/media/HBByBGDasAAZuAA?format=jpg&name=medium)







](/ceo_tommy1/article/2022246626765734324/media/2022241595819732992)

Starter Storyの「事例データベース」画面。 誰が、どんなビジネスで、毎月いくら稼いでいるかが、包み隠さず公開されています。

これらは「読み物」ではありません。「データ」です。なぜ、読者は「個人の面白い記事」よりも「無機質なデータ」にお金を払うのでしょうか？

-   再現性があるから： 一人の天才の意見より、100人の凡人のデータの方が参考にしやすい。
    
-   検索できるから： 「自分と同じ状況（例：資金ゼロ、副業）」の事例だけを探せる。
    
-   嘘がないから： 「こうすれば稼げる」という扇動ではなく、「こうして稼いだ」という事実しかない。
    

記事は古くなれば読まれませんが、データは古くなれば「歴史」になり、数が集まれば「統計」になります。

-   「ECサイトの平均利益率は？」
    
-   「キャンドル販売で成功した人の共通点は？」
    

彼が持っているのは、記事ではなく「検索できる資産」です。

労働集約型のライターを卒業し、情報が積み上がるほど価値が増す側に回ったのです。

> トミー： 多くの人は「毎日更新」という労働に逃げます。しかし、本当にやるべきは「積み上がる仕組み」の設計です。明日には価値がなくなる記事を書くのはやめて、10年後も検索されるデータを貯めてください。

# 

■ 第2章：編集者になるな、工場の「ライン長」になれ

「4,000人も取材するのは無理だ」 そう思いますか？

確かに、普通に「対面取材」をしていては不可能です。 だからこそ、彼はこの工程から「職人的なこだわり」を排除し、「工程の標準化」を行いました。

彼の取材プロセスは、工場のラインと同じように設計されています。 インタビューはZoomなどの同期コミュニケーションに依存せず、テキストベースで完結するように作られています。

![画像](https://pbs.twimg.com/media/HBBzcy5akAAMVZ_?format=jpg&name=large)

実際のインタビュー記事画面。この長文コンテンツはPat自身が書いたものではありません。質問フォームを送るだけで、起業家たちが「宣伝のために」自ら喜んで執筆してくれるのです。

## 

【Starter Storyの標準質問リスト】

-   概要： あなたは誰で、どんなビジネスをしていますか？（売上・利益の開示）
    
-   背景： バックグラウンドは？ どうやってアイデアを思いつきましたか？
    
-   検証： 最初の商品をどうやって形にしましたか？
    
-   集客： ローンチ以来、顧客を集めるために何が最も効果的でしたか？
    
-   ツール： 業務に使用しているツールやソフトウェアは？
    

彼は記事を書きません。「相手が答えるべき項目」を用意し、そこにデータを流し込んでもらうパイプラインを構築したのです。

これは手抜きではなく、「標準化」です。 質問が統一されているからこそ、読者は複数の事例を横並びで比較できます。

「クリエイター」としての情緒を捨て、「システム設計者」に徹したからこそ実現できた規模です。

> トミー： クオリティにこだわって1記事に3日かける職人は、ビジネスの世界では負けます。60点の記事を大量生産できるラインを作った人間が、市場を押さえるのです。「質」は後からついてきます。まずは「量」を担保する仕組みを作ってください。

# 

■ 第3章：SEOの概念を変える「プログラマティックSEO」

インタビュー記事が100件、200件と集まってきた頃、Pat Wallsはあることに気づきました。

> 「せっかく面白い事例があるのに、読者は『新着記事』しか見てくれない…」

過去の貴重な記事が埋もれていくのはもったいない。 そこで彼は、集まったインタビュー記事に「タグ」を貼り始めました。

-   起業家Aさんの記事（キャンドル販売） → タグ：「キャンドル」「女性起業家」「資金10万円以下」
    
-   起業家Bさんの記事（石鹸販売） → タグ：「石鹸」「副業」「資金5万円」
    
-   起業家Cさんの記事（キャンドル販売） → タグ：「キャンドル」「夫婦経営」
    

そして、この「タグ」がついた記事を、自動でひとまとめにするページを作りました。すると、どうなったか？ 彼が寝ている間に、システムが勝手に「新しいまとめ記事」を生成し始めたのです。

## 

やっていることは「Excelのフィルター」と同じ

難しく考える必要はありません。 あなたがExcelで「商品リスト」を管理していると想像してください。

「キャンドル」でフィルターをかければ、キャンドルの一覧が出ます。「資金5万円」でフィルターをかければ、低資金の一覧が出ます。

彼はこれと同じことをWebサイト上でやっただけです。

## 

【人間（ブログ）の限界】

あなたが「キャンドルビジネスの始め方」という記事を書くなら、情報を集めて執筆するのに5時間はかかるでしょう。

書き終わった後、もし読者が「石鹸ビジネス」も知りたがっていたら？またゼロから5時間労働です。

## 

【Pat Walls（コード）の革命】

彼は記事を書きません。ただ、インタビューに「タグ」を貼っただけです。 すると彼が作ったプログラムが、

-   「キャンドル」タグの記事を集めて ⇒ 「キャンドルビジネスの始め方」ページを自動生成
    
-   「資金5万円」タグの記事を集めて ⇒ 「資金5万円でできる起業アイデア集」ページを自動生成
    

このように、「切り口（タグ）」を変えるだけで、無数の「まとめ記事」を一瞬で量産できるシステムを作り上げたのです。

これが、彼が勝った本当の理由です。

特別な魔法を使ったわけではありません。「情報を整理するルール（タグ）」を決めただけなのです。

> トミー：「そうは言っても、それをサイトにする技術がない」と思いましたか？ 確かに2018年の当時は、彼は自分でプログラムを書く必要がありました。 しかし今は、NotionやWordPress、あるいはNoCodeツールを使えば、誰でも「タグで管理するサイト」を作れます。

重要なのは「技術」ではなく、「どうタグ付けして管理するか」という設計思想です。ここさえあれば、ツールは何でもいいのです。

■ 第4章：これは「ビジネス系」だけの話ではない。全業界で通用する「型」

「でも、これはビジネスジャンルだからできたんでしょ？」

そう思った方へ。この「コンテンツは書かずに、データベースを作る」というモデルは、あらゆるジャンルで再現されています。

あなたが参入しようとしている業界でも、同じことが可能です。

【共通点】

彼らは全員、「何を書こうか？」と悩むのではなく、「世の中に散らばっている何のデータを整理すれば、価値が出るか？」を考えた。

ただそれだけで、個人の限界を超えた収益とポジションを手にしているのです。

> トミー： あなたの業界にも、必ず「散らばっていて見にくいデータ」があります。 「キャンプ場の電源有無」「フリーランスの実際の請求書」「美容整形のダウンタイム経過写真」。 これらを整理するだけで、あなたは今日からその業界の「第一人者」になれます。書く必要はありません。集めて並べるだけです。

# 

■ 第5章：なぜ、あなたの売上は「月50万」で止まるのか？

ここまで読んで、「よし、自分もデータベースを作ろう」と思ったなら、少し待ってください。 その前に、あなた自身が直面している「ある事実」と向き合う必要があります。

もしあなたが今、副業や起業で「月30万〜50万」稼げているなら、あなたは間違いなく優秀です。 しかし、同時にこう感じていませんか？

-   「これ以上稼ごうと思ったら、寝る時間を削るしかない」
    
-   「売上は上がったけど、自由な時間が全くない」
    

なぜ、今の延長線上で「月300万、500万」の世界に届かないのか？ 答えはシンプルです。あなたが「労働」で稼いでいるからです。

-   月50万のモデル（あなた）： 記事を書く、動画を撮る、SNSを投稿する。 手が止まれば、売上も止まる。
    
-   月500万のモデル（Pat Walls）： 仕組みを作る、データを流し込む、タグ付けする。 寝ている間も、勝手に価値が増える。
    

「データベース型」の本質は、メディア運営ではありません。

> 「自分の労働を切り離しても、価値が積み上がる構造」

を持つことです。、ここには最大の壁があります。

-   「構造を変えるための『コンセプト』が作れない」
    
-   「今の忙しい作業を回しながら、新しい仕組みを作る時間がない」
    

今の労働を止めるわけにはいかない。そして、新たに月300万への種まきを余裕もない。 このジレンマこそが、99%の起業家が「小金持ち」で終わる原因です。

> トミー：労働量を10倍に増やして、売上を10倍にするのは不可能です。売上の「桁」を変える唯一の方法は、「構造」を変えることです。 Pat Wallsは「書く」のをやめて「設計」しました。あなたも今すぐ、プレイヤーから「設計者」に回るべきです。

# 

■ 結論：「設計者」になるための時間を、AIで買う

ここから導き出される答えはシンプルです。

「今の労働を続けながら、裏で『資産型モデル』を構築しろ」

しかし、ゼロからコンセプトを考え、LPを作り、人を集める仕組みを作るには、膨大な時間がかかります。だからこそ、私はこの「ローンチ特化型AI」を開発しました。

これは、あなたの代わりに「資産型ビジネスの設計図」を一瞬で書き上げるシステムです。

## 

【このAIツールが提供する「3つの武器」】
