# 副業プロジェクト方向性の議論メモ（2026-03-09）

## 背景・目的

- 本業の給料が不足しており、副業で収入を補いたい
- **目標**: 月5万円を安定的に稼ぎ、Claude Code subscription費用をまかないつつ、画像生成AI等への課金で収益化の幅を広げる
- **使えるツール**: Claude Code, Gemini CLI, Google Antigravity（AIのみ）
- **制約**: 副業に割ける時間は **1時間/日**
- **収益化手段**: note記事 + X投稿（文章ベース）

## 現状

- noteフォロワー: **ゼロ**（スタート前）
- 公開済み記事: **なし**
- 記事ストック: 約10本（経済指標3、株分析2、資産形成3、週次レポート1）
- 自動化基盤: RSS収集・AI要約・GitHub Issue管理・記事生成ワークフロー整備済み

## noteの収益モデル

| 手段 | 単価目安 | 月5万に必要な規模 |
|------|---------|-------------------|
| 有料記事（単品） | 300-500円 | 月100-170人購入 |
| 定期購読マガジン | 500-1000円/月 | 50-100人の定期読者 |
| メンバーシップ | 500-1000円/月 | 同上 |
| サポート（投げ銭） | 不安定 | 予測困難 |

## 方針: リーンスタートアップ的アプローチ

既存の `generate-market-report` 等は手動チェック工数が大きく、現段階では集客手段として非効率。
**テーマの制限を外し、内容を細かく見る必要がないテーマを複数並列で走らせ、反応をテストする。**

### Phase 1: テスト期間（最初の1-2ヶ月）

**目標**: 週5-7本を無料で公開し、どのテーマにPV・スキが集まるか検証

### Phase 2: 絞り込み（2-3ヶ月目）

- PV・スキのデータを見てテーマを2-3に絞る
- 好評テーマを深掘りシリーズ化
- 有料記事・マガジンのテスト開始

## テーマの分類（チェックコスト順）

| レベル | テーマ例 | チェック不要な理由 |
|--------|---------|-------------------|
| **極低** | 投資用語解説、投資の心理学、行動経済学コラム | 一般的知識。間違っても実害が小さい |
| **低** | 資産形成の基本（複利、積立の考え方）、家計管理 | 原理原則ベース。時事性なし |
| **低** | 投資本の要約・書評、名言・格言解説 | 元ネタが固定されている |
| **中** | セクター解説（半導体とは、AI産業構造） | 構造的な話。日々の値動きに依存しない |
| **高** | 個別銘柄分析、マクロ経済予測、週次レポート | ファクトチェック必須。誤情報リスク大 |

テスト期間は **上3つ（極低〜低）** を中心に進める。

### テーマ候補（並列実行）

1. **「投資のキホン」シリーズ** — 複利、ドルコスト平均法、分散投資など
2. **「投資の心理学」シリーズ** — 損失回避、アンカリング、FOMO等
3. **「用語解説」シリーズ** — PER、PBR、利回りなど1記事1用語
4. **「資産形成ロードマップ」** — 年代別、目的別の資産形成ガイド
5. **「投資本レビュー」** — 有名投資書籍のエッセンス紹介

### 1日の作業フロー（1時間）

- 10分: テーマ選定・AI生成指示
- 5分: 生成された記事をざっと確認
- 5分: note投稿 + X投稿
- 残り: 翌日以降分のストック生成

## AI臭さ問題への対策

### 原因

1. **没個性** — 誰が書いても同じ内容になる
2. **教科書的** — 正しいが面白くない
3. **体温がない** — 「私」が見えない

### 対策6つ

#### 1. 自分の実体験・数字を入れる（最重要）

```
× 「分散投資は重要です。リスクを低減できます」
○ 「月3万円を3年積み立てた結果、○○万円になった。
   暴落時に-20%食らったが、積立を続けた理由は…」
```

工数: 1記事あたり2-3行。5分で済む。

#### 2. 「意見・立場」を明確にする

```
× 「インデックス投資と個別株投資にはそれぞれメリットがあります」
○ 「私は個別株をやめた。その理由を3つ書く」
```

#### 3. 「マリーさんトーン」の活用

プロジェクト既存の `mary-tone-writer` スキルを活用。感情豊かな文体でAI臭さを軽減。

#### 4. フォーマットを工夫する

| AI臭いフォーマット | 人間臭いフォーマット |
|---|---|
| ○○とは？メリット3選 | 「○○を始めて失敗した話」 |
| 初心者向けガイド | 「過去の自分に教えたい3つのこと」 |
| ○○の比較表 | 「○○と△△、どっちを選んだか、なぜか」 |

#### 5. 「AI + 人間」のハイブリッド戦略を明示する

AI活用を隠さず、むしろコンテンツにする。
例: 「AIに○○を聞いてみた結果、人間が補足すべき点」

#### 6. X投稿との連動で「人格」を作る

Xでの日常的な発信（相場への反応、感想）が人格を形成。記事はその延長として読まれる。

### ワークフローへの実装案

1. **体験談プレースホルダー** — 生成時に `[ここに自分の体験を入れる]` を挿入
2. **意見・立場の指定** — プロンプトに立場を必須パラメータ化
3. **タイトルのフォーマット制約** — 「○○とは」「○○のメリット」を禁止、ストーリー型を強制
