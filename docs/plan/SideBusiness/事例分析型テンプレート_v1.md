# 事例分析型テンプレート v1

**作成日**: 2026-03-09
**対象テーマ**: 副業・資産形成（婚活は対象外 → 合成パターン法テンプレートv2を使用）

---

## 共通ルール

### 文字量

| 項目 | 基準 |
|------|------|
| 目標文字量 | 6,000-8,000字（データカード除く） |
| 読了時間の目安 | 電車内10-15分 |
| 最低文字量 | 5,500字（これ以下はリジェクト） |

### 視点・トーン

- **分析者視点**（三人称）で書く。一人称の架空体験は使わない
- 「〜した人がいる」「〜という事例がある」のようにソースを明示する書き方
- データや数字を多用し、主張には根拠を添える
- マリーさんトーンの活用OK（感情豊かな「です・ます」調 + 独自の視点）

### 埋め込みリンク方針

- `[EMBED]` マーカーのあるセクションに1-3個の実在リンクを配置
- リンク先: 元ソース（Reddit投稿、note記事、ブログ、YouTube等）
- リンクの前後に**分析文脈**を書く（「この事例が示しているのは…」）

### 肉付け指針（事例分析型）

1. **数字の具体性**: 「稼いだ」ではなく「月3.2万→6ヶ月後に月8万」
2. **比較と対比**: 成功事例 vs 失敗事例、通説 vs 実態
3. **読者の状況に引き寄せる**: 「会社員で週10時間しかない人にとっては…」
4. **再現性の評価**: この成功は再現可能か？条件は何か？
5. **読者への問いかけ**: セクション末尾に共感ポイントや行動促進の問いかけ

## テンプレートA: ジャンル別事例分析

**用途**: 特定の副業ジャンルや投資手法にフォーカスした記事
**例**: 「Webライティング副業で月5万達成した人のリアルな戦略」「つみたてNISAで3年続けた人の運用実績まとめ」

### データカード（記事冒頭・非公開メタデータ）

```
【記事メタ】
- テーマ: 副業 | 資産形成
- ジャンル: （例: Webライティング / 動画編集 / つみたてNISA / 高配当株）
- 分析対象事例数:
- ソース: （例: Reddit r/freelanceWriters, note.com #Webライター, Web検索）
- ターゲット読者: （例: 副業未経験の会社員、投資初心者）
- 記事タイプ: ジャンル別事例分析

【キーワードタグ】
# 例: #副業 #Webライティング #月5万 #成功事例 #会社員
```

### セクション構成

```
## タイトル: [ジャンル名]で[成果]を達成した人の[切り口]

### 導入 — なぜ今[ジャンル]なのか
目安: 600-1,000字
ポイント: 市場規模・トレンドデータで関心を引く。読者の「自分にもできるか？」に応える導入。身近なエピソードやSNSの声も交えて臨場感を出す。

### 事例紹介 — 実際にうまくいった人たち
目安: 2,000-2,800字
ポイント: 3-5件の事例を紹介。各事例は「属性→やったこと→結果」の3行サマリー + 2-3段落の深掘り。数字の推移、具体的な行動、ターニングポイントを丁寧に描写する。
[EMBED] 元ソースへのリンク（Reddit投稿、ブログ記事、note記事等）

### パターン分析 — 成功者に共通する戦略
目安: 1,500-2,000字
ポイント: 事例から抽出した3-5個の共通パターン。各パターンに具体的な事例を根拠として添える。通説との違いや意外性があれば強調する。

### 落とし穴 — よくある失敗と回避法
目安: 800-1,200字
ポイント: 失敗事例や挫折パターンも紹介。成功事例との対比で学びを深める。「自分もやりがちだな」と思わせる共感ポイントを入れる。
[EMBED] 失敗から学べるリソース

### ロードマップ — 始めるなら何から？
目安: 800-1,200字
ポイント: 1ヶ月目→3ヶ月目→6ヶ月目の具体的ステップ。必要な初期投資・時間も明示。各ステップで「ここでつまずきやすい」注意点も添える。
[EMBED] おすすめリソース（学習教材、プラットフォーム等）

### まとめ — このジャンルの現実的な期待値
目安: 400-700字
ポイント: 「誰でも簡単」ではなく、現実的な期待値を提示。読者への問いかけで締める。
```

## テンプレートB: ジャンル横断共通点分析

**用途**: 複数ジャンルを横断して成功の共通パターンを抽出する記事
**例**: 「副業で月5万超えた人に共通する5つのマインドセット」「投資初心者が1年で100万貯めた人の共通ステップ」

### データカード

```
【記事メタ】
- テーマ: 副業 | 資産形成
- 分析対象ジャンル: （例: Webライティング, 動画編集, プログラミング等を横断）
- 分析対象事例数:
- ソース:
- ターゲット読者:
- 記事タイプ: ジャンル横断共通点分析

【キーワードタグ】
# 例: #副業 #成功法則 #共通点 #月5万
```

### セクション構成

```
## タイトル: [対象]で[成果]を達成した人に共通するN個の[切り口]

### 導入 — 分析の概要
目安: 600-1,000字
ポイント: 「○件の事例を分析した結果、N個の共通点が見えてきた」というフレーミング。どんな人たちの事例を集めたか、分析の切り口を明示。

### 共通点1: [パターン名]
目安: 1,000-1,400字
ポイント: パターンの説明 + 2-3件の具体的事例。数字で裏付け。なぜこのパターンが効くのかの理由分析も添える。
[EMBED] 代表的な事例ソース

### 共通点2: [パターン名]
目安: 1,000-1,400字
（同上）

### 共通点3: [パターン名]
目安: 1,000-1,400字
（同上）
[EMBED] 代表的な事例ソース

### （共通点4-5: 必要に応じて追加。各800-1,000字）

### 意外だった点 — 通説との違い
目安: 800-1,200字
ポイント: 「〜と思われがちだが、実際は…」の形で発見を共有。読者の固定観念を揺さぶる。具体的な反例を挙げて説得力を持たせる。

### まとめ — 明日からできるアクションプラン
目安: 500-800字
ポイント: 共通点を踏まえた具体的な行動ステップ。読者への問いかけ。
```

## テンプレートC: AI一人スタートアップ事例分析（看板シリーズ）

**用途**: AI活用で成功した一人起業家/小規模チームの戦略分析
**例**: 「AIで月商$10万を達成した一人起業家の戦略分析」「一人でAIプロダクトを作って$1M ARR達成した3つの事例」

### データカード

```
【記事メタ】
- テーマ: 副業（看板シリーズ）
- 分析対象事例数:
- ソース: （例: Indie Hackers, Hacker News, Reddit r/SideProject, r/Entrepreneur, X）
- 地域: （例: 米国, 欧州, グローバル）
- ターゲット読者: AI活用に関心のある副業・起業志望者
- 記事タイプ: AI一人スタートアップ事例分析

【キーワードタグ】
# 例: #AI #一人起業 #スタートアップ #海外事例 #SaaS #月商100万
```

### セクション構成

```
## タイトル: [人物/プロダクト名]が[成果]を達成した戦略を分析する

### 導入 — AI一人スタートアップの波
目安: 600-1,000字
ポイント: なぜ今AIで一人起業が可能になったのか。市場の変化、象徴的なデータや発言を引用。読者の「自分にも関係あるのか？」に応える。

### 事例深掘り（1-3件）
目安: 各1,200-2,000字（合計2,500-4,000字）
各事例で以下を分析:
- **プロダクト概要**: 何を作ったか、誰に売っているか
- **ビジネスモデル**: 収益構造（SaaS/マーケットプレイス/API等）
- **AI活用のポイント**: どこにAIを使い、何が人間の判断か（ツール名列挙ではなく活用戦略）
- **成長の軌跡**: ローンチ→初収益→スケールの数字付きタイムライン
- **成功要因**: なぜこの人はうまくいったのか
- **苦労した点**: 順風満帆ではなかった部分も描写し、リアリティを持たせる
[EMBED] プロダクトサイト、Indie Hackers投稿、HNスレッド等

### パターン分析 — AI一人起業で成功する人の共通項
目安: 1,200-1,800字
ポイント: 事例横断で見えてくる共通パターン。「技術力」だけではないビジネスセンスの部分を重点的に。成功者が共通して「やらなかった」ことにも着目。

### 日本市場への示唆 — このモデルは日本で再現できるか
目安: 800-1,200字
ポイント: 海外事例をそのまま紹介で終わらせず、日本市場での応用可能性を分析。言語・文化・市場規模の違いを考慮。既に日本で類似の取り組みがあれば紹介。
[EMBED] 参考になる日本語リソース

### まとめ — 一人でも戦える時代に何を選ぶか
目安: 400-700字
ポイント: 煽りではなく現実的な視点で締める。読者への問いかけ。
```

## テンプレート選択ガイド

| 書きたい記事 | テンプレート |
|-------------|-------------|
| 特定の副業ジャンルの成功事例 | A: ジャンル別事例分析 |
| 特定の投資手法の成功事例 | A: ジャンル別事例分析 |
| 複数ジャンルの成功共通点 | B: ジャンル横断共通点分析 |
| AI活用の海外起業事例 | C: AI一人スタートアップ事例分析 |
| 婚活体験談 | 体験談DB統一テンプレートv2（合成パターン法） |

## ソース収集の違い

| テンプレート | 主要ソース | 言語 |
|-------------|-----------|------|
| A (ジャンル別) | Reddit日英, note.com, Web検索 | 日本語中心 |
| B (横断分析) | 複数ジャンルのソースを統合 | 日本語中心 |
| C (AI起業) | Indie Hackers, HN, Reddit r/SideProject | **英語中心** |

## Neo4j連携

事例分析型は以下のノード構造で管理:

```
(:CaseStudyArticle {article_id, theme, article_type, title, wordcount_target, wordcount_current, status})
  -[:ANALYZES]-> (:SuccessCase {case_id, source_url, summary, genre, outcome})
  -[:BELONGS_TO]-> (:CandidateTheme)

(:Memory:project {name: '副業プロジェクト'})
  -[:HAS_ARTICLE]-> (:CaseStudyArticle)
```

### ID命名規則

| ノード | ID形式 | 例 |
|--------|--------|-----|
| CaseStudyArticle | `csa-{theme}-{sequential}` | `csa-sidehustle-001` |
| SuccessCase | `sc-{genre}-{sequential}` | `sc-webwriting-001` |
