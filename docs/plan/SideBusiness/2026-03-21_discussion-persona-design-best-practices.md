# 議論メモ: 転職アカウント ペルソナ設計 & ベストプラクティス

**日付**: 2026-03-21
**参加**: ユーザー + AI
**Neo4j Discussion ID**: disc-2026-03-21-persona-design-best-practices
**前提議論**: disc-2026-03-21-asp-research

## 背景・コンテキスト

転職アカウントの具体的なペルソナ設計にあたり、ユーザーの要望:
- 他のユーザーを自分のアカウントに注目させたい
- アフィリエイト臭がしないようにしたい
- 本質的に他のユーザーの役に立つ情報を発信したい
- 同じような収益化で成功している例を調査してベストプラクティスをまとめる

## 調査結果: 6つのベストプラクティス

### 原則1: 投稿比率の黄金律（7:3ルール）

リンク入り（収益化）投稿は全体の**2〜3割**に抑える。残り7〜8割は純粋な価値提供に徹する。
全投稿にリンクを貼るとThreadsアルゴリズムから評価が下がり、インプレッションが激減（シャドウバン）する。

### 原則2: 投稿の型 — 結論→ストーリー→問いかけ

```
結論: 「転職面接で一番聞かれるのは志望動機じゃない」
    ↓
ストーリー: 「自分が3社受けて気づいたのは...」（実体験・失敗談）
    ↓
問いかけ: 「みんなが面接で一番困った質問って何？」
```

返信率が劇的に上がり、アルゴリズム評価が上昇。

### 原則3: ファン化の4ステップ

| ステップ | やること | 転職垢での具体例 |
|---------|---------|---------------|
| 1. 共感 | プロフィールで過去の悩みを見せる | 「ブラック企業3年→転職で年収1.5倍にした人」 |
| 2. 信頼 | 失敗談を隠さず実体験で語る | 「1社目の転職は大失敗だった話」 |
| 3. 関係性 | 他アカウントへのリプライで対話 | フォロワーの転職相談に丁寧に返信 |
| 4. 導線 | プロフィール→固定投稿→リンク | 自然な流れで転職サービス紹介ページへ |

### 原則4: アフィリエイトリンクの自然な配置

**NG**: 「おすすめの転職サイトはこちら→[リンク]」（唐突）

**OK**: 投稿本文で問題→解決のストーリーを語る → コメント欄で「ちなみに自分が使ったサービスはこれ。無料登録で書類添削もしてくれるのが良かった → [リンク] PR」

### 原則5: コミュニティファースト（R.E.A.L.戦略）

Sendible社の実績: 300 views/四半期 → 172,000 views（**30,000%成長**）

| 要素 | 意味 | 実践方法 |
|------|------|---------|
| R | Reliable（信頼） | 一貫したテーマで毎日投稿 |
| E | Empowering（力づけ） | フォロワーの行動を後押しする情報 |
| A | Authentic（本物） | 失敗も含めた正直な発信 |
| L | Led by community | 自分の投稿だけでなく他者の投稿にリプライして会話に参加 |

### 原則6: ツリー投稿で滞在時間を伸ばす

1投稿に長文を詰め込まず、返信形式でスレッドを繋げる。ユーザーの滞在時間が伸び、アルゴリズム評価が上がる。

## 転職アカウントのペルソナ設計

**ポジション**: 「転職のプロ」ではなく「転職で失敗も成功もした先輩」

**コンテンツ構成比**:
- 有益投稿（70%）: 面接Tips、職務経歴書の書き方、年収交渉術、転職あるある
- エンゲージメント投稿（20%）: 「転職で一番辛かったこと教えて」等の問いかけ
- 収益化投稿（10%）: 実体験ベースのサービスレビュー + コメント欄PRリンク

**差別化**: 自動化インフラによる毎日の一貫した投稿（手動運用者には真似できない量と一貫性）

## 決定事項

1. **dec-2026-03-21-content-ratio-rule**: 投稿比率7:3ルール（有益70%/エンゲージメント20%/収益化10%）を採用
2. **dec-2026-03-21-persona-positioning**: ペルソナは「失敗も成功もした先輩」、R.E.A.L.戦略でコミュニティファースト運用
