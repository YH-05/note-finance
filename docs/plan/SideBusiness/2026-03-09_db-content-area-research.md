# 調査メモ: DB型コンテンツアプローチの情報エリア調査

**日付**: 2026-03-09
**参加**: ユーザー + AI

## 背景

トミー(@ceo_tommy1)の記事「寝ながら月1,000万」で紹介されたStarter Story（Pat Walls）のデータベース型コンテンツモデルを、本プロジェクトに応用する方法を検討。DB型アプローチと相性の良い情報エリアを網羅的に調査。

## 調査ソース

- プログラマティックSEO成功事例（英語圏20+事例）
- 日本のDB型サイト個人開発事例（indieverse.co.jp等）
- note.com有料コンテンツ動向分析（2026年1月発表データ）
- note.com収益化パターン分析

## Tier 1: 最有望（6候補）

### A. 資産形成パターンDB
- **DB型**: 体験談型（Starter Story型）
- **概要**: 年齢/年収/家族構成/ポートフォリオ/利用制度/資産推移を統一フォーマットで構造化
- **データソース**: Reddit/note/X上の既存体験談を構造化
- **差別化**: DB形式では存在しない。「自分と同じ境遇」を見つける需要
- **note相性**: 有料マガジン/メンバーシップ向き
- **note成長率**: 中
- **構造化容易性**: ◎
- **データ入手性**: ○
- **リスク**: 低

### B. 副業体験談DB
- **DB型**: 体験談型（Starter Story型）
- **概要**: 副業種類×収入×時間投資×成功のコツを統一フォーマットで構造化
- **データソース**: Reddit/note/X上の既存体験談を構造化
- **差別化**: noteの急成長カテゴリ（複業・在宅ワーク）に直接合致。DB形式は少ない
- **note相性**: 実用ノウハウ系で平均価格1,842円
- **note成長率**: 高（+179%）
- **構造化容易性**: ◎
- **データ入手性**: ◎
- **リスク**: 低

### C. 転職体験談DB
- **DB型**: 体験談型（Starter Story型）
- **概要**: 業界×年齢×年収変化×活動期間×使ったサービスを統一フォーマットで構造化
- **データソース**: Reddit/note/X上の既存体験談を構造化
- **差別化**: 大手はメディア型。リアル体験談のDB形式は少ない
- **note相性**: キャリア系はnoteで人気
- **note成長率**: 中
- **構造化容易性**: ◎
- **データ入手性**: ◎
- **リスク**: 低

### D. 恋愛・婚活体験談DB ★追加調査で特定
- **DB型**: 体験談型（Starter Story型）
- **概要**: 年齢×性別×年収×地域×使ったサービス×活動期間×費用×結果を統一フォーマットで構造化
- **データソース**: Reddit(r/dating_advice等)/note/X上の既存体験談を構造化
- **差別化**: 婚活体験談は個別ブログやXに散在するが、DB形式で横比較できるものは存在しない。「自分と同じ年齢・年収・地域の成功パターン」を探す需要が極めて高い
- **note相性**: HARMの法則Relationship。感情価値高く有料購入意欲も高い
- **note成長率**: 最高（6.3倍）— 全カテゴリ中1位
- **構造化容易性**: ◎
- **データ入手性**: ○
- **リスク**: 低

### E. 住宅購入/不動産体験談DB ★追加調査で特定
- **DB型**: 体験談型（Starter Story型）
- **概要**: 年齢×年収×家族構成×地域×物件タイプ×価格×ローン条件×後悔ポイント×満足度を統一フォーマットで構造化
- **データソース**: Reddit/note/マンションコミュニティ等
- **差別化**: SUUMOの口コミはあるが、購入判断プロセス全体をDB形式で横比較できるものは存在しない
- **note相性**: HARMの法則Money。資産形成パターンDBとの相乗効果
- **note成長率**: 中
- **構造化容易性**: ○
- **データ入手性**: △（体験談がオープンに共有されにくい）
- **リスク**: 低

### F. メンタルケア/回復体験DB ★追加調査で特定（条件付き）
- **DB型**: 体験談型
- **概要**: 職業×年齢×症状×きっかけ×対処法×回復期間×費用×現在の状態を統一フォーマットで構造化
- **データソース**: Reddit/note上の既存体験談を構造化
- **差別化**: noteが最も高く評価する「再現不能な個人体験」そのもの。DB形式は存在しない
- **note相性**: 「メンタルケア・生き方」は売れる7ジャンルの1つ。HARMの法則Health
- **note成長率**: 高
- **構造化容易性**: ○
- **データ入手性**: △
- **リスク**: 中-高（医療的アドバイスと誤解されるリスク。免責事項設計必須）

## 重要な発見

### note.com急成長データ（2026年1月発表）
- メンバーシップ売上: 前年比+81.3%
- 急成長カテゴリ: AI活用 / SNS運用 / 育児 / 複業・在宅ワーク
- 実用ノウハウ系の平均価格: 1,842円（読み物系の約2倍）
- 文字数と売上の相関: ほぼゼロ

### 個人DB型サイト成功者の教訓
> 「市場選定が全て。競合が手動では原価構造的に無理で、法人もDB化していないところを狙え。大きく作る前に小さく検証。」
> — indieverse.co.jp

### note.com上でのDB型実現方法
1. 統一フォーマット記事の蓄積 → マガジンでカテゴリ化
2. 定期まとめ記事をAIアシストで生成
3. メンバーシップで全DBアクセス権を月額課金化
4. X連携でSNS流入（note集客の70%がSNS）

## Tier 1にならなかった候補と理由

| 候補 | 却下理由 |
|------|---------|
| 育児パターンDB | 子供の個性依存が大きく「パターン」化困難。切り口を絞れば可能性はある |
| グルメMAP DB | すでに高単価の成功事例が存在（東京グルメMAP 2,980円×419名）。後発は差別化困難 |
| フリーランス独立DB | 副業体験談DB(B)と重複が大きい |
| 留学/海外移住DB | noteでの需要規模が小さい |
| クレカ比較DB | 単なる情報列挙（スペック比較）であり体験談型ではない |
| 投資本レビューDB | 同上。レビュー集約は体験談の構造化ではない |
