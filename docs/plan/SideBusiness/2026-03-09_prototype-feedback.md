# 議論メモ: 試作フィードバックと改善方針の決定

**日付**: 2026-03-09
**参加**: ユーザー + AI
**ステータス**: 完了

## 背景・コンテキスト

3テーマ（婚活・副業・資産形成）の試作パターン各1件が完成した段階で、ユーザーから4つのフィードバックを受領。
試作の方向性は肯定的だが、文字量・実体験感・システム基盤に改善が必要。

## ユーザーフィードバック

1. **全体的に試作はいい感じ** → 方向性維持
2. **文章量が足りない** → 電車内10分で読める程度に拡張が必要
3. **実体験が見えてこない** → 実在する情報（YouTube動画、ブログ、店舗等）をnoteの埋め込みリンクとして配置する提案
   - 例: ダイエット動画を見る→痩せる→婚活成功（動画リンク埋め込み）
   - 例: 資産形成ブログを見る→知識を得る→暴落でも平常心（ブログリンク埋め込み）
   - 例: このお店でデート成功（お店リンク埋め込み）
4. **TODO 4件** → システム改善の要望

## 決定事項

### 1. 埋め込みリンク方針: ポイント埋め込み

- 「転機」「きっかけ」等の重要セクションのみ1-2個の実在リンクを配置
- 全セクションに入れるのではなく要所に集中
- テンプレートに `[EMBED]` マーカーを追加

### 2. 目標文字量: 6,000-8,000字

- 現在の試作（2,500-3,200字）の約2倍
- 肉付けポイント:
  - 各セクションの感情描写・内面の葛藤を深掘り
  - サブエピソード（友人との会話、SNSで見た投稿への反応等）
  - 数字データの文脈解説を厚く
  - 埋め込みリンク周辺の体験ストーリー（「この動画を見て何が変わったか」）
  - 読者への問いかけや共感ポイント

### 3. TODO着手順序

| 順位 | TODO | 理由 |
|------|------|------|
| 1 | グラフDB蓄積基盤 | 他の全TODOの土台。ソース情報のNeo4j蓄積 |
| 2 | マルチエージェント批評 | 初稿→複数AI批判→改善。既存finance批評を転用可能 |
| 3 | 実体験組合せロジック | 体験チェーン自動生成。差別化に直結 |
| 4 | 知識接続サイクル | 蓄積知識の定期リレーション更新 |
