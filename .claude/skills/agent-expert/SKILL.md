# Agent Expert

Claude Code エージェントの設計・実装・検証に関するナレッジベーススキルです。

## 目的

このスキルは以下を提供します：

- **設計原則**: 高品質なエージェントを作成するための設計ガイドライン
- **テンプレート**: 標準的なエージェント構造のテンプレート
- **検証ルール**: フロントマターと内容の品質検証基準
- **カテゴリ分類**: エージェントタイプの使い分けガイド

## リソース

詳細ガイドと参照資料:

| ファイル | 内容 |
|---------|------|
| `guide.md` | 設計原則・エージェントカテゴリ・協調パターン |
| `template.md` | エージェント定義ファイルのテンプレート構造 |
| `frontmatter-review.md` | フロントマター検証ルールと品質チェックリスト |

## エージェントタイプ

### 1. オーケストレーター型

複数のサブエージェントを制御するリーダーエージェント。

```yaml
color: blue
model: opus
```

### 2. スペシャリスト型

特定ドメインのタスクを実行する専門エージェント。

```yaml
color: purple
model: sonnet
```

### 3. チームメイト型

Agent Teams ワークフローの一員として動作するエージェント。

```yaml
color: green
model: inherit
```

## フロントマター必須フィールド

| フィールド | 必須 | 説明 |
|-----------|------|------|
| `name` | ✅ | エージェントのスラッグ（kebab-case） |
| `description` | ✅ | 一行の説明（トリガー判定に使用） |
| `model` | ✅ | `inherit`, `sonnet`, `opus`, `haiku` |
| `color` | 推奨 | `gray`, `blue`, `green`, `purple`, `red`, `yellow` |

## 完了条件

- [ ] `guide.md` を参照して設計原則を確認
- [ ] `template.md` のテンプレートに従って実装
- [ ] `frontmatter-review.md` で品質検証を実施
- [ ] エージェントタイプ（オーケストレーター/スペシャリスト/チームメイト）が明確
