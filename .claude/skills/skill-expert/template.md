# スキルテンプレート

## ナレッジベース型 SKILL.md テンプレート

```markdown
---
name: my-skill-name
description: |
  [スキルの概要説明（1-2文）]
  [いつプロアクティブに使用するか]
  Use PROACTIVELY when [条件].
allowed-tools: Read
---

# [スキル名]

[スキルの概要説明]

## 目的

このスキルは以下を提供します：

- **[機能1]**: [説明]
- **[機能2]**: [説明]

## いつ使用するか

### プロアクティブ使用（自動で検討）

以下の状況では、ユーザーが明示的に要求しなくても参照：

1. **[ユースケース1]**
   - 「[キーワード]」
   - 「[キーワード]」

2. **[ユースケース2]**
   - [状況説明]

### 明示的な使用

- 「[スキル名]を参照して」という直接的な要求

## [メインコンテンツセクション]

[スキルの核心となるガイドライン・パターン・参考情報]

## 関連リソース

| ファイル | 内容 |
|---------|------|
| `guide.md` | 詳細ガイドライン |
```

## ワークフロー型 SKILL.md テンプレート

```markdown
---
name: my-workflow-skill
description: |
  [ワークフローの概要（1文）]
  /[command-name] コマンドで使用。
  [自動実行の条件があれば記載]
allowed-tools: Read, Write, Edit, Bash, Glob, Grep, Task, AskUserQuestion
---

# [スキル名]

[スキルの概要説明]

## 使用方法

```bash
/[command-name] [引数]
/[command-name] --option value
```

## ワークフロー

```
Phase 1: [フェーズ名] ─── [出力]
    |
Phase 2: [フェーズ名] ─── [出力]
    |
Phase 3: [フェーズ名] ─── 完了
```

## [フェーズ詳細]

### Phase 1: [フェーズ名]

[具体的な処理内容]

## 完了条件

- [ ] [条件1]
- [ ] [条件2]
```

## サブファイル テンプレート（guide.md）

```markdown
# [スキル名] ガイド

## 詳細手順

### 手順1: [タイトル]

[詳細説明]

### 手順2: [タイトル]

[詳細説明]

## トラブルシューティング

### 症状: [問題]

**原因**: [原因説明]

**解決策**:
1. [手順1]
2. [手順2]
```
