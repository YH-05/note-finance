# 分析の深さ 評価基準

## 概要

事例分析型記事の核心的価値は「分析」にある。
単なる事例の列挙・紹介ではなく、事例間の比較やパターン抽出、因果メカニズムの解明が求められる。

## カテゴリ別基準

### 1. パターン抽出（pattern_extraction）

#### チェック項目

- 複数事例を横断した共通パターンが明示されているか
- 各パターンに具体的な事例が根拠として添えられているか
- パターンの数が適切か（3-5個が目安）
- 表面的な共通点（「努力した」「諦めなかった」）ではなく、具体的・行動可能な共通点か

#### スコアリング

| 状態 | スコア |
|------|--------|
| 独自性のあるパターンが根拠付きで提示 | 90-100 |
| パターンが明示され、根拠もある | 70-89 |
| パターンはあるが根拠が薄い | 50-69 |
| パターン抽出がない/表面的 | 0-49 |

### 2. 因果推論（causal_reasoning）

#### チェック項目

- 「なぜうまくいったか」のメカニズムが説明されているか
- 相関と因果を区別しているか
- 反事実的思考（「もし～だったら」）が含まれているか
- 外部環境要因（市場タイミング、技術トレンド等）も考慮されているか

#### 深さのレベル

| レベル | 例 |
|--------|-----|
| 表面 | 「Aさんは成功した。なぜなら努力したから」 |
| 中程度 | 「Aさんの成功要因はニッチ市場の選択にある。競合が少ない分野で…」 |
| 深い | 「Aさんが選んだ市場は競合3社だったが、既存ソリューションがSMB向けに最適化されておらず、その隙間を突いた。同時期にAPI基盤が成熟したことも追い風に…」 |

### 3. 比較対比（comparison）

#### チェック項目

- 成功事例 vs 失敗事例の対比があるか
- 通説 vs 実態の対比があるか（「～と思われがちだが、実際は…」）
- 事例間の比較（異なるアプローチの比較）があるか
- 日本市場 vs 海外市場の比較（テンプレートCの場合）があるか

### 4. 独自洞察（insight）

#### チェック項目

- 他の類似記事にはない視点が含まれているか
- 読者の「意外！」という反応を引き出せる発見があるか
- 既存の常識を覆す or 補完する洞察があるか
- 「成功者がやらなかったこと」への着目があるか

## 記述 vs 分析の判定基準

| 種別 | 例 |
|------|-----|
| 記述（事実の報告） | 「AさんはAI SaaSを作り、月商$10万に達した」 |
| 分析（解釈・推論） | 「Aさんの成功は、技術力よりも市場選択に起因する。なぜなら…」 |

推奨比率: 記述40% : 分析60%（分析セクションにおいて）
