# データ正確性 評価基準

## 概要

事例分析型記事で引用される数字・事実・ソースの正確性を評価する。
事例分析型は「信頼できる分析」が価値であるため、データの正確性は最重要観点の1つ。

## カテゴリ別基準

### 1. 数値裏付け（numerical）

#### チェック項目

- 金額（収益、ARR、MRR等）にソース or 時期が明示されているか
- 成長率・倍率の計算が正しいか（例: 月3万→月8万 を「2.7倍」と書いているか）
- 「約」「推定」等の適切なヘッジングがあるか
- 複数ソースで数字が矛盾していないか

#### スコアリング

| 状態 | スコア |
|------|--------|
| 全数値にソースor根拠あり | 90-100 |
| 大半にソースあり、一部不明 | 70-89 |
| ソースなしの数値が多い | 50-69 |
| 数値の誤りが検出される | 0-49 |

### 2. ソース明示（source）

#### チェック項目

- 事例の出典が明記されているか（Indie Hackers投稿、HNスレッド、ブログ記事等）
- 「～という事例がある」「～と報告されている」等の帰属表現があるか
- [EMBED] マーカー位置に実際のリンク候補が想定されているか
- 二次ソース経由の情報を一次ソースとして扱っていないか

#### スコアリング

| 状態 | スコア |
|------|--------|
| 全事例にソース明示 | 90-100 |
| 大半にソースあり | 70-89 |
| ソース不明の事例が混在 | 50-69 |
| ソースがほぼ示されていない | 0-49 |

### 3. 時系列整合（timeline）

#### チェック項目

- 事例内の時系列が矛盾していないか（ローンチ→初収益→スケールの順序）
- 市場データの時期と事例の時期が整合しているか
- 「最近」「近年」等の曖昧な時期表現が多用されていないか

### 4. 誇張検出（exaggeration）

#### チェック項目

- 「必ず」「絶対」「確実に」等の断定表現の頻度
- 成功事例のみの紹介になっていないか（生存者バイアス）
- 「誰でもできる」「簡単に」等の安易な表現
- 適切なヘッジング（「多くの場合」「条件次第で」等）の有無

#### 禁止語（事例分析型固有）

| 禁止 | 推奨 |
|------|------|
| 誰でもできる | 条件を満たせば再現可能性がある |
| 簡単に稼げる | 一定の努力と時間を投資すれば |
| 必ず成功する | 成功事例に共通するパターンは |
| 絶対おすすめ | 分析結果から有望と考えられる |
