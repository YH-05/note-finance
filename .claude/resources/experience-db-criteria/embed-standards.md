# 埋め込みリンク適切性基準（共通）

## 概要

記事内の埋め込みリンク（EmbeddableResource）が、体験の文脈に自然に組み込まれ、
読者体験を向上させているかを評価する基準。

## 前提: ポイント埋め込み方針

- `[EMBED]` マーカーのあるセクションのみ、1-2個の実在リンクを配置
- 全セクションに入れるのではなく**要所に集中**
- リンクの前後に**体験文脈**を必ず書く

## 評価項目

### 1. 配置の自然さ（placement）

埋め込みリンクが記事の流れを中断していないか。

**良い配置**:
- 体験の中で「このリソースに出会った」瞬間に配置
- 転機セクションの「何がきっかけで変わったか」の具体例として
- 試行錯誤期の「この情報が助けになった」の文脈で

**悪い配置**:
- 唐突に「おすすめリソースはこちら」と挿入
- 冒頭に列挙（まだ文脈がない状態で）
- 末尾にまとめて列挙（体験と切り離されている）

**チェック項目**:
- [ ] リンクが体験の時系列の中に自然に配置されているか
- [ ] リンクの前に「なぜこのリソースに辿り着いたか」が書かれているか
- [ ] リンクの後に「見た/読んだ結果、何が変わったか」が書かれているか
- [ ] テンプレートの `[EMBED]` マーカー位置と一致しているか

### 2. 前後の体験文脈（context）

リンクの前後に十分な体験ストーリーがあるか。

**必須構成**:
```
[体験の文脈: なぜこのリソースに辿り着いたか]
  ↓
[埋め込みリンク]
  ↓
[影響の描写: 見た/読んだ結果、何が変わったか]
```

**良い例**:
```
その夜、YouTubeで「つみたてNISA 初心者」と検索した。
出てきた動画を片っ端から再生した。

[EMBED: YouTube動画]

全部の動画が同じことを言っていた。「長期・分散・積立」。
父のFXとは根本的に別物だった。
```

**悪い例**:
```
参考になる動画があります。

[EMBED: YouTube動画]

次のセクションに進みます。
```

**チェック項目**:
- [ ] リンク前に100字以上の体験文脈があるか
- [ ] リンク後に100字以上の影響描写があるか
- [ ] 「このリンクを見て何が変わったか」が明確か

### 3. テーマとの関連性（relevance）

埋め込みリソースが記事のテーマ・セクション内容と合致しているか。

**チェック項目**:
- [ ] リソースのジャンルが記事テーマと一致しているか
- [ ] リソースの難易度が記事のターゲット読者に合っているか
- [ ] Neo4j の EmbeddableResource の `embed_section` と実際の配置セクションが一致しているか
- [ ] リソースが実在し、URLが有効か（リンク切れチェック）

### 4. リソース種類の多様性（variety）

同じ種類のリソースに偏っていないか。

**リソース種類**:
- YouTube動画
- ブログ記事 / note記事
- 書籍
- サービス / アプリ
- 店舗 / 施設

**チェック項目**:
- [ ] 1記事内で同じ種類のリソースが3つ以上連続していないか
- [ ] 最低2種類以上のリソース種類が使われているか
- [ ] 動画とテキストなど、メディア形式のバリエーションがあるか

## Neo4j との整合性チェック

スキルが Neo4j から取得した EmbeddableResource の情報と記事内容を照合:

- `resource_type` と実際のリンク形式が一致しているか
- `embed_section` と実際の配置セクションが一致しているか
- `experience_context` と記事内の体験文脈が整合しているか
- `is_active` が true のリソースのみ使用されているか
