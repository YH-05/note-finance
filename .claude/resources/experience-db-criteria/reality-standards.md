# リアリティ基準（共通）

## 概要

体験談の「本当にありそう」「嘘っぽくない」度合いを評価する基準。
合成パターン法で作られた記事が、実体験として自然に読めるかを判定する。

## 評価項目

### 1. 五感描写（sensory）

体験の場面に五感の描写が含まれているか。

**良い例**:
- 視覚: 「東京駅の中央線ホームで電車を待っていた。隣にはカップルが立っていて」
- 聴覚: 「あの通知音がなくなるだけで、こんなに気が楽になるとは」
- 触覚: 「設定完了ボタンを押したとき手が震えていた」
- 味覚/嗅覚: 「一人でお湯を沸かしてカップ麺を食べながら」

**悪い例**:
- 「婚活を始めることにした」（場面がない）
- 「副業で苦労した」（抽象的）

**チェック項目**:
- [ ] 各セクションに最低1つの五感描写があるか
- [ ] 「転機」セクションに特に豊かな場面描写があるか
- [ ] 抽象的な感情（辛い、嬉しい等）が具体的場面に置き換えられているか

### 2. 数字・固有名の具体性（specificity）

数字や固有名詞が、帯域化ルールの範囲内で具体的に使われているか。

**良い例**:
- 「手取り22万から家賃7万、奨学金2.5万、光熱費・通信費で2万」
- 「Pairs → with → Omiai（並行利用期間あり）」
- 「マッチ数（累計）: 約120人」

**悪い例**:
- 「給料から色々引かれて残りが少ない」（数字がない）
- 「あるマッチングアプリを使った」（サービス名がない）

**チェック項目**:
- [ ] 金額はスペックカードの帯域と整合しているか
- [ ] 具体的なサービス名・プラットフォーム名が使われているか
- [ ] 時間経過が具体的か（「しばらく後」ではなく「3ヶ月目の終わり」）

### 3. 心理変化の段階性（psychology）

心理の変化が唐突でなく、段階的に描かれているか。

**良い例**:
- 焦り → 期待 → 失望 → 消耗 → 休止 → 内省 → 再開 → 成功
- 不安 → 挑戦 → 挫折 → 気づき → 転機 → 成長

**悪い例**:
- 「辛かったけど頑張ったら成功した」（段階がない）
- 急に前向きになる（転機の描写なし）

**チェック項目**:
- [ ] ネガティブ→ポジティブの間に「葛藤」のフェーズがあるか
- [ ] 転機が内的な気づきと外的なきっかけの両方で描かれているか
- [ ] 成功後も「完全にハッピー」ではなく、残る課題や不安が書かれているか

### 4. スペックカードとの整合性（consistency）

本文の内容がスペックカードのデータと矛盾していないか。

**チェック項目**:
- [ ] 年齢・性別の表現が一貫しているか
- [ ] 金額がスペックカードの帯域内か
- [ ] 期間がスペックカードと一致しているか
- [ ] 結果がスペックカードの記載と一致しているか
- [ ] キーワードタグと本文の内容が対応しているか
