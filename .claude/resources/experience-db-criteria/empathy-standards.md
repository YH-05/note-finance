# 共感度基準（共通）

## 概要

読者が「自分のことのように」感じられるか、感情移入して最後まで読み通せるかを評価する基準。
note.com のスマホ読者（通勤中・隙間時間）が離脱しないことが最重要。

## 評価項目

### 1. 感情の起伏・弧（emotion_arc）

記事全体を通じて感情のアップダウンが設計されているか。

**理想的な感情弧**:
```
共感（冒頭） → 不安・焦り → 失敗・挫折（谷）→ 小さな気づき → 転機（山）→ 成長 → 余韻
```

**チェック項目**:
- [ ] 冒頭2-3段落で読者の共感を掴んでいるか
- [ ] 明確な「谷」（最も辛い場面）があるか
- [ ] 「谷」から「転機」への移行に説得力があるか
- [ ] 末尾に余韻（読後に考えさせる要素）があるか
- [ ] 感情が単調にならず、緩急があるか

**減点パターン**:
- 冒頭がスペックカードの説明で始まる（感情のフックがない）
- ずっとネガティブで「谷」が見えない
- 急に成功して感情の移行が不自然

### 2. 読者の自己投影しやすさ（identification）

読者が「この人は自分と似ている」と感じられるか。

**効果的なテクニック**:
- 具体的だが限定的すぎない属性描写
- 「あるある」エピソード（多くの人が経験する場面）
- 内面の独白（「〜と思った」形式）

**良い例**:
- 「飲み会1回のリカバリーに1週間かかる生活」（多くの若手社会人が共感）
- 「画面を閉じた後のリビングの静けさが、前より少しだけ重く感じる」

**悪い例**:
- 特殊すぎる環境の描写（読者が自分に当てはめられない）
- 理想的すぎる人物像（「年収1000万だけど孤独」等、共感しにくい）

**チェック項目**:
- [ ] ターゲット読者層が自分を重ねられる属性設定か
- [ ] 3つ以上の「あるある」エピソードが含まれているか
- [ ] 内面の独白が適切な頻度で挿入されているか

### 3. 弱さの開示・正直さ（vulnerability）

失敗、恥ずかしさ、弱さを正直に書けているか。

**効果的な開示**:
- 「正直に言えば焦っていた」
- 「深夜3時にクラウドワークスに登録した。プロフィールはほぼ空欄」
- 「計算すると胸が痛くなったから、途中でやめた」

**避けるべきパターン**:
- 失敗を美化する（「あの経験があったからこそ今がある」だけ）
- 弱さを見せずに成功だけ語る
- 自虐が過剰（読んでいて辛くなる）

**チェック項目**:
- [ ] 「恥ずかしい」「情けない」と感じた場面が率直に書かれているか
- [ ] 失敗の描写が具体的で、取り繕っていないか
- [ ] 弱さの開示と強さのバランスが取れているか（自虐一辺倒でない）

### 4. 普遍的な感情への接続（universality）

個別の体験が、より大きな普遍的テーマに接続しているか。

**普遍的テーマの例**:
- 「自分は何者なのか」というアイデンティティの問い
- 「知らなかったことのコスト」
- 「愛情に基づいた助言と正しい助言は違う」
- 「消去法でも、選んだことに意味がある」

**チェック項目**:
- [ ] 体験の教訓が読者自身の人生にも当てはまる形で書かれているか
- [ ] 「振り返り」セクションが単なるTipsではなく、深い気づきを含んでいるか
- [ ] テーマ固有の体験が、テーマを超えた共感を生んでいるか
