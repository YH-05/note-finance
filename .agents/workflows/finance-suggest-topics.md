# 金融記事トピック提案

## パラメータ（ユーザーに確認）

| パラメータ | デフォルト | 説明               |
| ---------- | ---------- | ------------------ |
| カテゴリ   | 全カテゴリ | 特定カテゴリに限定 |
| count      | 5          | 提案数             |

## 処理フロー

### 1. 既存記事の確認

// turbo

```bash
ls -d articles/*/ 2>/dev/null
```

各記事の `article-meta.json` から topic, category, created_at を取得し、カテゴリ分布を集計する。

### 2. トピック提案の生成

以下の評価基準でスコアリングした提案リストを生成する:

| 基準            | 高スコア（8-10）    | 中スコア（5-7） | 低スコア（1-4） |
| --------------- | ------------------- | --------------- | --------------- |
| timeliness      | 今週のイベント      | 今月のイベント  | 時事性なし      |
| information     | データ豊富          | 十分なデータ    | データ不足      |
| reader_interest | SNS話題、検索需要高 | 一定の関心      | ニッチ          |
| feasibility     | 3000-5000字で完結   | 調整必要        | 大幅調整必要    |
| uniqueness      | 独自視点あり        | 標準的          | 類似記事多数    |

### 3. 結果の表示

以下の形式で表示:

```
## トピック提案

### 既存記事
- 総数: N件
- カテゴリ分布

### 提案トピック

#### 1. [トピック名]
- カテゴリ: [category]
- スコア: XX/50
- 対象: [シンボル]
- 期間: [date_range]
- 提案理由: [reason]
- キーポイント: [key_points]

### カテゴリバランス分析
不足しているカテゴリがあれば提案
```

### 4. 次のアクション案内

選択されたトピックで記事を作成する場合:

- `/new-finance-article` ワークフローへ誘導

## 関連ワークフロー

- `/new-finance-article` - 記事フォルダ作成
- `/finance-full` - 記事作成の全工程
