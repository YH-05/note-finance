# 金融記事トピック提案スキル

金融コンテンツの読者層に響く、時事性と独自性の高いトピックをデータに基づいて提案します。

## 推奨ワークフロー

1. **既存記事の分析**: 
   `scripts/analyze_existing_articles.py` を実行し、既存記事のカテゴリ分布と最新の投稿内容を確認します。

2. **外部トレンドの調査**: 
   以下のMCPツールを組み合わせて、現在のマーケットの関心事を収集します。
   - **Reddit (`reddit-finance-topics` スキルなど)**: 投資コミュニティの最新の議論やセンチメントを把握。
   - **RSS (`rss` MCP)**: 登録済みの金融ニュースフィードから最新のヘッドラインを取得。
   - **SEC EDGAR (`sec-edgar-mcp`)**: 個別銘柄に言及する場合、直近の 8-K や 10-Q の重要イベントを確認。
   - **Wikipedia**: トピックの背景知識の補完。

3. **トピックの生成とスコアリング**:
   [scoring_criteria.md](references/scoring_criteria.md) の基準（時事性、情報量、関心度、実現可能性、独自性）に基づき、各50点満点で評価します。

4. **提案の提示**: 
   以下のフォーマットでトピックを提示します。

```markdown
## 既存記事の状況
- 総記事数: N件
- カテゴリ分布: [例: 米国株: 40%, マクロ: 30%, ...]

## 提案トピック

### 1. [トピックのタイトル]
- **カテゴリ**: [category]
- **スコア**: XX / 50 (時事性: X, 情報量: X, 関心度: X, 実現性: X, 独自性: X)
- **提案理由**: [なぜ今、このトピックなのか]
- **構成の骨子**:
  - [キーポイント1]
  - [キーポイント2]
- **推奨ツール**: [例: SEC EDGARで8-Kを確認、Redditの反応を引用など]
```

## 次のステップへの案内

トピックが決まったら、`/new-finance-article` （新規記事フォルダ作成）や `/finance-full` （記事作成）スキルの利用を促してください。
